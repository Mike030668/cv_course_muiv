# Публикация решений после дедлайна

1. Храним решения локально/в приватном облаке (Google Drive, S3, etc.).
2. После дедлайна: публикуем `solutions_release/<hw>` с эталонными ноутбуками и краткими разбором.
3. Можно приложить `Release` на GitHub, либо отдельную ветку `solutions/`.

> Автоматизация: допускается GitHub Actions по расписанию (cron) с загрузкой артефактов из приватного хранилища.
